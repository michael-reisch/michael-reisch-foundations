CSS Concept Research - display: inline; vs display: inline-block;

To properly explain this concept, we must first explore the CSS box model. Basically, the box model consists of a box object in the body of a website. This box object consists of the content, surrounded by padding, surrounded by border, surrounded by margin.

Every html element has a default display property. The display property determines the element layout behaviour in relation to other elements on the page. For example, <div>, <p> and <h1> (among other) elements are block-level elements. This means that the element's box will start on their own line, and take up the entire width of that line. It's important to note that you can adjust height and width properties of block-level elements. Block-level elements will push other elements onto another line. There are numerous different display values that you can use to manipulate how each element flows within the layout of your web page. The values we will be looking at in this blog post are inline and inline-block.

Inline

Some elements that have display: inline; as their default property are: <span>, and <label>. Inline elements will take up only as much space as they need, will not push other elements to new lines (so they will display side-by-side), and do not accept width, height or top-bottom margin properties. However, they will accept padding and left-right margin properties. Inline-Block

There are not too many html elements with inline-block as their default display property. But some that do are: <img> and <button>. Inline-block acts kind of like a hybrid of block and inline display properties. They are similar to block-level elements in that you can adjust the height, width and margin properties. And they are similar to inline elements in that they can be placed next to eachother on the same line.
